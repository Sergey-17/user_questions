
Подготовь полный деплой для сервера, на котором мы сейчас есть, с автоматическим обновлением контейнеров из Docker Hub.

Условия:

Есть два сервиса: backend и frontend.
Docker Hub ник: sergey17.
Docker-compose должен:
Создавать отдельные контейнеры backend и frontend.
Использовать один общий bridge network, чтобы они могли общаться друг с другом точно так же, как на локальной машине.
Поднимать сервисы в фоновом режиме (docker-compose up -d).
Напиши bash-скрипт 'update-containers.sh", который делает следующее:
Проверяет новые версии образов на Docker Hub (sergey17/backend:latest и
sergey17/frontend:latest).
Если есть новые версии, делает pull, останавливает текущие контейнеры и поднимает
новые версии через docker-compose up -d.
Выводит лог действий: какие контейнеры обновлены, какие оставлены без изменений.
Настрой cron jоb, который запускает этот скрипт каждый час.
Добавь комментарии к каждому шагу скрипта и docker-compose, чтобы можно было
легко менять пути, имена контейнеров и теги.
Учти, что на сервере Docker и docker-comроse уже установлены, bash доступен, WSL не нужен.
